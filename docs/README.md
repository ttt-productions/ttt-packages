# ttt-packages docs

Documentation for the `@ttt-productions/*` shared monorepo.

## Folders

### `packages/`
Per-package reference documentation. One file per package describing what it contains, its API surface, key types, and design decisions specific to that package. **Read these to use a package.**

### `design/`
Cross-cutting design rules and architectural invariants that span multiple packages or govern how packages must be built. Server-safe main entry, identity invariants, upload path shape, etc. **Read these to design a package change.**

### `playbooks/`
Audit methodologies and recurring checks. Grep-based scripts and step-by-step procedures for verifying packages still follow the rules. **Read these to audit packages.**

### `audits/`
One-off audit output files. Generated by playbook runs. Not committed long-term — delete after the underlying issue is resolved.

## Where to start

- New to a package → read `packages/<name>.md`.
- Designing a change that touches package boundaries or affects how packages are consumed → read everything in `design/`.
- Verifying a package still follows the rules → run the relevant playbook in `playbooks/`.
