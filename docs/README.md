# ttt-packages docs

Documentation for the `@ttt-productions/*` shared monorepo.

## Folders

### `packages/`
Per-package reference documentation. One file per package describes what that package owns, what it must not own, and which entry points are public. Read these when using or changing a package.

### `design/`
Cross-cutting design rules and architectural invariants that span multiple packages or govern how packages must be built. Server-safe entries, React boundaries, identity invariants, upload path shape, and similar rules live here.

### `playbooks/`
Audit methodologies and recurring checks. Grep-based scripts and step-by-step procedures for verifying packages still follow the rules live here.

### `generated/`
Auto-generated reference, rebuilt from source (e.g. the Firestore schema generated from `COLLECTION_SCHEMAS`). Never hand-edit — run the generator and commit the result.

### `audits/`
One-off audit output files. Generated by playbook runs. Do not treat these as permanent source-of-truth docs; delete or archive them after the underlying issue is resolved.

## Where to start

- New to a package → read `packages/<name>.md`.
- Designing a change that touches package boundaries or affects how packages are consumed → read the relevant files in `design/`.
- Verifying a package still follows the rules → run the relevant playbook in `playbooks/`.
